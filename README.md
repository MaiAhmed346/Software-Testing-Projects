# Gennie Application – Software Testing Project

## 📌 Overview
This project demonstrates my practical experience in software testing by applying test design and execution on a sample application (Gennie App) and API endpoints.

The goal of this project is to simulate real-world QA activities including test case creation, validation, and defect identification.

---

## 🎯 Scope of Testing
- Functional Testing
- API Testing
- Negative Testing
- Test Case Design

---

## 🧪 Test Activities Performed

### 1. Test Case Design
- Created structured test cases covering main user flows
- Included positive and negative scenarios
- Validated input fields and system behavior

### 2. API Testing
- Tested API endpoints using Postman
- Validated response status, structure, and data consistency
- Checked edge cases and invalid inputs

### 3. Defect Identification
- Simulated bug scenarios based on unexpected system behavior
- Documented issues with clear steps, expected vs actual results

---

## 🧰 Tools Used
- Postman
- Excel / Documentation
- JIRA (conceptual understanding)

## 📊 Key Scenarios Covered
- User data validation
- API response validation
- Error handling for invalid inputs
- System behavior under different conditions
